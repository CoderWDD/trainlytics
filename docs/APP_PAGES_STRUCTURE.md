# Trainlytics 应用页面结构规划

## 页面层级概览

```
应用壳层（App Shell）
├── 底部导航栏（4个主要入口）
│   ├── 今日（Today）
│   ├── 历史（History）
│   ├── 分析（Insights）
│   └── 模板（Templates）
├── 全局快速添加（Quick Add FAB）
└── 设置入口（顶部溢出菜单）
```

---

## 一、应用壳层（App Shell）

### 功能组件
- **顶部应用栏**
  - 页面标题
  - 日期上下文
  - 溢出菜单（设置、导出等）

- **底部导航栏**
  - 今日（Home图标）
  - 历史（Calendar图标）
  - 分析（Chart图标）
  - 模板（Template图标）

- **全局快速添加按钮（FAB）**
  - 悬浮在右下角
  - 在活跃训练时隐藏
  - 点击打开快速添加底部弹窗

---

#### 功能列表
1. **每日概览**
   - 显示当前日期
   - 显示阶段标签（减脂/增肌/维持）
   - 显示完成度摘要
   - 切换表单模板入口

2. **身体状态管理**
   - 记录/编辑体重（主要）
   - 记录/编辑体脂率
   - 记录/编辑腰围
   - 记录自定义恢复指标（睡眠、步数、心率等）
   - 点击卡片展开详情

3. **餐食管理**
   - 显示4个餐食槽位（早/午/晚/零食）
   - 每个餐食卡片显示：
     - 时间
     - 热量和蛋白质
     - 完成状态
     - 模板来源标记
   - 点击添加/编辑餐食
   - 展开查看食物明细

4. **训练管理**
   - 空状态：显示"开始训练"按钮
   - 进行中：显示会话摘要和"继续训练"按钮
   - 已完成：显示训练摘要和亮点

5. **每日笔记**
   - 折叠状态显示预览
   - 展开后显示完整编辑器
   - 记录饥饿感、能量水平、日程变化等

#### 交互行为
- 各区域支持展开/折叠
- 空白区域优先展开
- 已有数据区域默认显示摘要
- 向下滚动查看所有区域
- 点击任意卡片进入编辑模式

## 十一、AI餐食识别功能详细设计

### 11.1 功能概述

AI餐食识别功能通过计算机视觉和深度学习技术，让用户只需拍照即可快速记录餐食营养信息，大幅降低录入门槛。

### 11.2 识别流程

```
用户拍照/选择图片
    ↓
图片预处理（压缩、裁剪）
    ↓
上传到AI服务
    ↓
AI模型分析
    ├─ 食物检测（位置、边界）
    ├─ 食物分类（种类识别）
    ├─ 重量估算（基于参照物）
    └─ 营养计算（查询数据库）
    ↓
返回识别结果
    ↓
用户确认/调整
    ↓
保存到本地数据库
```

### 11.3 AI识别能力

**支持识别的食物类型**
- 主食类：米饭、面条、面包、馒头、饺子等
- 肉类：鸡胸肉、牛肉、猪肉、鱼肉、虾等
- 蔬菜类：西兰花、生菜、番茄、黄瓜等
- 蛋奶类：鸡蛋、牛奶、酸奶、奶酪等
- 水果类：苹果、香蕉、橙子等
- 其他：豆腐、坚果、调味品等

**识别精度**
- 食物种类识别准确率：≥90%
- 重量估算误差：±15%
- 营养成分计算误差：±10%

### 11.4 用户交互设计

**拍照界面**
- 实时相机预览
- 网格辅助线（帮助构图）
- 闪光灯控制
- 参照物提示（如：放置硬币、餐具作为尺寸参考）
- 多角度拍摄建议

**识别中状态**
- 进度条显示
- 预计等待时间（通常2-5秒）
- 可取消操作

**结果展示**
- 每个食物独立卡片
- 置信度显示（如：95%确定是鸡胸肉）
- 缩略图标注（在原图上框出识别区域）
- 一键全选/全不选
- 逐项编辑或删除

**错误处理**
- 识别失败提示（光线不足、图片模糊等）
- 未识别到食物时的引导
- 网络错误时的离线降级方案
- 重新拍照或切换手动输入

### 11.5 技术实现方案

**方案A: 云端AI服务（推荐）**
- 使用第三方AI API（如Google Cloud Vision、Azure Computer Vision、阿里云视觉智能）
- 优点：识别准确度高、模型持续更新、无需本地算力
- 缺点：需要网络连接、有API调用成本
- 实现：
  - 图片压缩到1MB以下
  - HTTPS上传到AI服务
  - 解析JSON结果
  - 映射到本地营养数据库

**方案B: 本地AI模型（可选）**
- 使用TensorFlow Lite或ML Kit
- 优点：离线可用、无API成本、隐私性好
- 缺点：识别准确度较低、模型体积大（50-200MB）
- 实现：
  - 集成轻量级食物识别模型
  - 本地推理
  - 定期更新模型

**混合方案（最佳实践）**
- 优先使用云端AI（准确度高）
- 网络不可用时降级到本地模型
- 用户可在设置中选择偏好

### 11.6 数据隐私

- 图片仅用于识别，不存储在云端
- 识别完成后立即删除服务器端图片
- 本地保存的照片加密存储
- 用户可选择是否保存原图
- 符合GDPR和隐私保护法规

### 11.7 营养数据库

**数据来源**
- USDA食物成分数据库
- 中国食物成分表
- 常见品牌食品数据
- 用户自定义食物库

**数据结构**
```
FoodDatabase {
  id: string
  name: string
  name_en: string
  category: string
  serving_size: number (g)
  calories: number (kcal/100g)
  protein: number (g/100g)
  carbs: number (g/100g)
  fat: number (g/100g)
  fiber: number (g/100g)
  aliases: string[] // 别名
  image_url: string
}
```

### 11.8 优化策略

**提高识别准确度**
- 用户反馈学习：用户修正后的数据用于优化
- 个性化识别：学习用户常吃的食物
- 时间上下文：早餐时段优先识别早餐食物
- 地域优化：根据用户地区优化食物库

**提升用户体验**
- 缓存常见食物数据（离线可查）
- 预加载相机权限
- 后台预热AI模型
- 识别历史快速重用

### 11.9 成本估算

**云端AI服务成本**
- Google Cloud Vision: $1.50/1000次
- Azure Computer Vision: $1.00/1000次
- 阿里云视觉智能: ¥0.0015/次

**假设场景**
- 每用户每天拍照3次
- 月活跃用户1000人
- 月识别次数：3 × 30 × 1000 = 90,000次
- 月成本：$90-135（约¥600-900）

**成本优化**
- 本地预筛选（过滤非食物图片）
- 批量识别折扣
- 缓存常见食物
- 用户自付费选项（高级功能）

### 11.11 替代方案（如果AI成本过高）

**语音输入**
- 用户说："一碗米饭，一块鸡胸肉，一份西兰花"
- 语音识别转文字
- NLP解析食物和份量
- 自动填充营养数据

**智能搜索**
- 快速搜索食物名称
- 自动补全常见食物
- 一键添加标准份量
- 比完全手动输入快3-5倍

**条形码扫描**
- 扫描包装食品条形码
- 查询品牌食品数据库
- 自动填充营养信息
- 适用于预包装食品

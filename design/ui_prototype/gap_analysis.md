# Trainlytics 设计稿缺页与遗漏分析

## 概述

本文档基于 README 中描述的核心功能特性，对现有 UI 设计稿（Figma）进行全面审查，识别缺失的页面、功能遗漏以及设计一致性问题。

---

## 1. 缺失页面

### 1.1 引导/欢迎页（Onboarding/Welcome）

**严重程度：高**

README 提及"减脂期"等场景，暗示应用面向有健身目标的用户。但设计稿中没有任何首次使用引导页面。新用户首次打开 App 后应看到：

- 品牌介绍与核心功能展示
- 基础身体数据采集（身高、体重、性别、年龄）
- 健身目标选择（减脂、增肌、维持、塑形等）
- 训练经验水平选择（新手、进阶、高级）
- 训练偏好（健身房/居家、可用器械等）

**影响**：没有引导页会导致新用户首次体验缺乏上下文，应用无法为用户提供个性化的初始数据。

---

### 1.2 目标设置页（Goal Setup）

**严重程度：高**

README 中明确提到"减脂期"等功能，但设计稿中没有专门的目标配置页面。用户需要能够设置和调整：

- 体重目标（目标体重、目标体脂率）
- 营养目标（每日热量、蛋白质/碳水/脂肪比例）
- 训练目标（每周训练次数、重点肌群）
- 目标周期（如"8周减脂计划"）

**影响**：没有目标设置，洞察模块的数据分析缺乏基准线，无法判断用户是否朝着目标方向前进。

---

### 1.3 个人资料/用户中心页（Profile Page）

**严重程度：中**

设置页面中隐含了个人资料入口，但没有独立的个人资料页。该页面应包含：

- 头像与用户名
- 身体基础数据（身高、年龄、性别）
- 当前身体指标（体重、体脂率、围度）
- 目标概览与进度
- 使用统计（累计训练天数、总训练次数等）

**影响**：用户无法在一个集中位置查看和管理自己的个人信息与目标进度。

---

### 1.4 训练完成总结页（Workout Completion Summary）

**严重程度：中**

在 `active_workout` 中点击"完成训练"后直接返回今日主页，缺少训练总结环节。完成训练后应展示：

- 本次训练总时长
- 总训练容量（总组数 × 重量）
- 各动作完成情况
- 达成的个人记录（PR）提示
- 与上次同类训练的简要对比
- 身体感受/疲劳度评估

**影响**：缺少训练总结会导致用户无法直观回顾训练成果，降低了完成训练后的成就感与正向反馈。

---

### 1.5 食物数据库浏览页（Food Database Browser）

**严重程度：低**

`meal_template_editor` 中的食物搜索为底部弹窗（`food_search_sheet`），缺少一个完整的食物数据库浏览页面。完整页面应提供：

- 食物分类浏览（肉类、蔬菜、主食、乳制品等）
- 热门/常用食物推荐
- 自定义食物管理
- 最近使用/收藏的食物列表

**影响**：弹窗式搜索在频繁添加食物时体验受限，完整页面能提供更好的浏览与发现体验。

---

### 1.6 数据导入页（Data Import）

**严重程度：低**

设置中仅有"导出数据"功能，缺少对应的数据导入页面。应支持：

- 从之前导出的 JSON 文件恢复数据
- 从其他健身 App 迁移数据（CSV 格式支持）
- 导入过程中的数据校验与冲突处理

**影响**：用户更换设备或重新安装后无法恢复历史数据。

---

### 1.7 通知/提醒设置页（Notification Settings）

**严重程度：低**

设置页面中未设计通知与提醒功能。应包含：

- 餐食记录提醒（早/午/晚餐提醒时间）
- 训练提醒（计划训练日提醒）
- 水分摄入提醒
- 周回顾提醒
- 静音时段设置

**影响**：缺少提醒机制会降低用户的使用黏性，容易忘记记录。

---

### 1.8 主题切换（Theme Toggle）

**严重程度：低**

设计稿中展示了深色模式界面，但没有浅色模式设计，也没有主题切换设置。应提供：

- 深色/浅色/跟随系统三种模式
- 浅色模式的完整视觉设计
- 切换入口（设置页面）

**影响**：仅支持深色模式可能在白天户外等场景下使用不便。

---

## 2. 功能遗漏

### 2.1 周回顾对比视图（Week Comparison）

`weekly_review` 页面仅支持单周查看，缺少周与周之间的对比功能。

**应增加**：
- 与上周/上周期的数据对比（体重变化、训练容量变化）
- 趋势箭头指示（上升/下降/持平）
- 目标达成率环比

---

### 2.2 训练日历/热力图（Training Calendar Heatmap）

`insights_dashboard` 仅有折线图展示趋势，缺少日历热力图视图。

**应增加**：
- 类似 GitHub 贡献热力图的训练日历
- 按训练强度/类型着色
- 月度/季度视图切换
- 快速跳转到某天的训练详情

---

### 2.3 身体数据趋势详情（Body Stats Detail Trend）

洞察模块有体重趋势图表，但缺少更详细的身体成分趋势分析。

**应增加**：
- 体脂率变化趋势
- 围度变化（腰围、臀围、胸围等）
- BMI 变化轨迹
- 多指标叠加对比视图
- 身体成分组成变化（肌肉量/脂肪量）

---

### 2.4 个人记录页（Personal Records Page）

没有专门的个人记录（PR）页面展示用户的最佳成绩。

**应增加**：
- 各动作最大重量记录
- 各动作最大容量记录
- PR 达成时间线
- PR 突破提醒/庆祝动效
- 按肌群/动作分类浏览

---

### 2.5 全局搜索（Global Search）

设计稿中没有全局搜索功能入口。

**应增加**：
- 搜索历史训练记录（按动作名、日期）
- 搜索历史餐食记录
- 搜索食物数据库
- 搜索模板
- 搜索入口可置于今日主页顶部或独立 Tab

---

### 2.6 关于页面（About Page）

设置页面隐含版本信息，但缺少独立的关于页面。

**应增加**：
- 应用版本号与更新日志
- 开源许可信息
- 隐私政策与用户协议
- 联系/反馈渠道
- 数据存储说明

---

## 3. 设计一致性问题

### 3.1 底部导航栏不一致

| 设计稿 | 底部导航 Tab | 语言 | Tab 数量 |
|--------|-------------|------|---------|
| `backfill_journal`（_8） | Dashboard, Journal, Analytics, Plan, Profile | 英文 | 5 个 |
| `ai_food_recognition` | DASHBOARD, WORKOUTS, NUTRITION, PROFILE | 英文（大写） | 4 个 |
| 其他所有页面 | 今日, 历史, 洞察, 模板 | 中文 | 4 个 |

**问题分析**：

1. **导航结构不同**：`backfill_journal` 使用 5 个 Tab（含 Dashboard 和 Plan），而主设计使用 4 个 Tab（今日、历史、洞察、模板）。功能覆盖有重叠但组织方式不同。

2. **语言混用**：部分页面使用英文大写标签，与主设计的中文风格不统一。

3. **命名差异**：同样的功能在不同设计稿中使用不同名称：
   - "今日" vs "Dashboard"
   - "洞察" vs "Analytics"
   - "模板" vs "Plan"

**建议**：统一为最终的 4 Tab 中文设计，并清理旧的迭代设计稿。

---

### 3.2 视觉风格不一致

- 部分弹窗使用圆角卡片样式，部分使用全屏页面
- `quick_add_sheet` 和 `start_workout_sheet` 均为底部弹窗，但视觉层级和交互方式未统一
- 颜色使用在不同页面间有细微差异（需核实设计系统是否完全对齐）

---

### 3.3 返回导航不统一

- 子页面完成后返回路径不明确：例如 `ai_food_recognition` 确认后返回哪里？
- `backfill_journal` 中的子页面返回后是否保留已填写的其他数据？
- 深层嵌套页面（如 `meal_template_editor` → `food_search_sheet` → `manual_food_entry`）的返回堆栈行为未定义

---

## 4. 建议优先级排序

### P0 — 必须补齐（阻塞核心体验）

| 优先级 | 缺失内容 | 原因 |
|--------|---------|------|
| P0-1 | 引导/欢迎页 | 新用户首体验的核心环节 |
| P0-2 | 目标设置页 | 所有数据分析和洞察的基础 |
| P0-3 | 训练完成总结页 | 训练流程的关键闭环 |
| P0-4 | 底部导航栏统一 | 必须在开发前解决的设计决策 |

### P1 — 强烈建议（显著提升体验）

| 优先级 | 缺失内容 | 原因 |
|--------|---------|------|
| P1-1 | 个人资料页 | 用户信息管理的核心入口 |
| P1-2 | 周回顾对比视图 | 让洞察数据更有意义 |
| P1-3 | 个人记录页 | 健身用户的核心驱动力 |
| P1-4 | 通知/提醒设置 | 提高用户留存率的关键 |
| P1-5 | 身体数据趋势详情 | 完善"减脂期"场景的数据支撑 |

### P2 — 建议补充（锦上添花）

| 优先级 | 缺失内容 | 原因 |
|--------|---------|------|
| P2-1 | 训练日历/热力图 | 可视化训练连贯性 |
| P2-2 | 全局搜索 | 提升历史数据查找效率 |
| P2-3 | 主题切换（浅色模式） | 扩大使用场景 |
| P2-4 | 数据导入页 | 数据安全与迁移保障 |
| P2-5 | 食物数据库完整页 | 提升餐食记录效率 |
| P2-6 | 关于页面 | 应用合规性需求 |

---

## 5. 总结

| 类别 | 数量 |
|------|------|
| 缺失页面 | 8 个 |
| 功能遗漏 | 6 项 |
| 设计一致性问题 | 3 类 |
| **P0 项** | **4 项** |
| **P1 项** | **5 项** |
| **P2 项** | **6 项** |

现有设计稿覆盖了 README 中大部分核心功能的 UI 表现，但在**新用户引导**、**目标管理体系**、**训练闭环体验**三个方面存在明显缺口。建议在进入开发阶段前，至少完成所有 P0 级别的设计补齐工作。
